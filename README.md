# IU-Startup
# ШколаПлан — автоматизированное составление школьного расписания

## Описание продукта

«ШколаПлан» — инструмент для автоматизированного составления школьного расписания с поддержкой индивидуальных учебных планов (ИУП) и деления на группы.

## Гипотеза

Внедрение интеллектуального модуля для работы с ИУП и делением на группы сократит время составления расписания с 14 дней до 1–2 дней и позволит найти реальных покупателей.

## Целевая аудитория

Завучи частных и государственных школ в возрасте 27–60 лет.

## Проблема клиента

Текущие процессы и ограничения:
* ручное распределение учеников по индивидуальным траекториям;
* существующие решения не справляются с пересечениями групп и ИУП;
* на составление расписания уходит от недели до двух;
* ошибки приводят к необходимости переделки всего расписания.

## Решение

Автоматический алгоритм «ШколаПлан»:
* загружает список учеников с выбранными предметами;
* учитывает кабинеты и учителей для каждой группы;
* поддерживает ИУП и разделение параллели по общим предметам;
* генерирует расписание за 10 минут;
* позволяет корректировать составленный план.

## Цели текущей версии

* подтвердить сокращение времени составления расписания до 1–2 дней (с учётом корректировки, целевое сокращение — 85 %);
* привлечь 5 пилотных школ;
* собрать данные для улучшения алгоритма.

## Метрики успеха

| Параметр | Способ измерения | Целевое значение |
|--------|---------------|-----------------|
| Время составления расписания | Сравнение времени до и после внедрения | Сокращение на 85 % (с 14 до 1–2 дней) |
| Количество итераций | Подсчёт числа корректировок после первого создания расписания | Минимизация количества итераций |
| NPS завучей | Опрос по шкале от 0 до 10 | > 8 из 10 |
| Готовность рекомендовать | Вопрос «Будете ли вы рекомендовать продукт коллегам?» | Ответ «да» от большинства респондентов |
| Готовность платить | Указание суммы в рублях, которую готовы заплатить за продукт | Сбор данных о приемлемом ценовом диапазоне |

## Минимальный функционал (MVP)

Обязательные функции:
* импорт учеников из Excel (ФИО + выбранные предметы);
* визуальный конструктор групп (drag‑and‑drop);
* проверка конфликтов в реальном времени;
* экспорт готового расписания в Excel.

Не включается в MVP:
* мобильные приложения;
* дополнительные интеграции;
* замена существующих систем.

## Подход к созданию дешёвого MVP (имитационное решение)

Этапы реализации:
1. Сбор требований от учителей через Google Формы.
2. Внесение данных в Excel и составление расписания с использованием макросов VBA.
3. Ручная доработка результатов (учёт особых случаев, корректировка конфликтов).
4. Визуализация результатов в удобном для заказчика виде (таблицы, графики, пояснения).
5. Предоставление результатов пилотным школам и сбор обратной связи.

## План действий

1. Разработать Google‑форму для сбора требований от учителей.
2. Создать шаблон Excel с макросами VBA для генерации расписания.
3. Провести пилотное тестирование с 1–2 школами.
4. Собрать обратную связь и доработать шаблон.
5. Привлечь ещё 3–4 школы для расширенного тестирования.
6. Проанализировать данные и сформировать план доработки полноценного алгоритма.

## Ожидаемые результаты пилотного запуска

* подтверждение гипотезы о сокращении времени составления расписания;
* получение отзывов от завучей о удобстве и полезности решения;
* сбор данных о наиболее частых конфликтах и сложностях при составлении расписания;
* формирование базы для разработки полноценного алгоритма.

## ОСНОВНЫЕ ФУНКЦИИ
**Функция 1. Импорт и «оживление» расписания**
* Методист загружает текущее расписание из Excel, CSV или копирует таблицу из браузера. Система парсит данные, распознаёт структуру (классы, предметы, учителя, кабинеты) и строит интерактивную модель школы. Умный парсер с нечётким сопоставлением: «Иванова М.П.» и «Иванова Мария Петровна» — один человек. Шаблоны импорта для популярных форматов (1С:АСР, Ректор, типовой Excel).
**Функция 2. СанПиН-аудит в один клик** КИЛЛЕР-ФИЧА
* После импорта методист нажимает «Проверить» и за 15 секунд получает полный аудит по 47 правилам СанПиН 1.2.3685-21. Результат — визуальный дашборд: каждый класс, каждый день, каждое правило. Зелёное — соответствует, жёлтое — рекомендация нарушена, красное — жёсткое нарушение. Отчёт со ссылками на пункты нормативов можно скачать в PDF и показать проверяющим.
**_Методист получает ценность за 30 секунд, ещё до того как что-либо оптимизировал. Это бесплатная функция и точка входа в продукт (PLG-воронка)._**
**Функция 3. Экстренные замены с AI-подбором**
* Завуч отмечает отсутствие учителя. Система за 5 секунд предлагает план замен — ранжированный список кандидатов для каждого урока. Для каждого варианта: ФИО, квалификация, текущая нагрузка, свободен ли в нужный слот, доступен ли кабинет, побочные эффекты («появится окно»). После подтверждения — готовая карточка замен (PNG/PDF) для стенда и чата в Сферуме.
Техническая основа: CP-SAT solver (Google OR-Tools) с warm start — фиксация неизменённых уроков, пересчёт только затронутых. Время от фиксации отсутствия до уведомления всех: 3 минуты вместо 45.
**Функция 4. Конструктор ИУП и автосборка групп**
Три шага.
* (1) Сбор выборов: система генерирует веб-форму, ученики выбирают предметы и уровни, система проверяет ФГОС-ограничения.
* (2) Кластеризация: из 120 индивидуальных наборов формируются 8–12 оптимальных учебных групп; методист видит тепловую карту и может корректировать drag-and-drop.
* (3) Генерация расписания для сформированных групп без конфликтов и с соблюдением СанПиН.
Результат: 1 день вместо 2–3 недель. Доля учеников с полностью бесконфликтным расписанием: >95%.
**Функция 5. Поиск слотов для проектных групп**
Руководитель проекта задаёт состав межклассной группы и требования (кабинет с проектором, 2 спаренных урока). Система находит общие свободные слоты, визуализирует доступность в виде тепловой карты и предлагает минимальные корректировки основного расписания при необходимости. Покрывает требование ФГОС по обязательной проектной деятельности.


## КЛИЕНТСКИЙ ПУТЬ
**Фаза 0. Обнаружение**
* Август, конец лета. Методист Елена Сергеевна ищет «составить расписание школа/проверка расписания СанПиН онлайн». Находит лендинг SmartSchedule: «Составьте расписание за 30мин/Загрузите расписание — получите аудит за 30 секунд. Бесплатно.»
**Фаза 1. Первая ценность за 2 минуты**
* Загружает Excel. Парсер распознаёт 38 классов, задаёт 2 уточняющих вопроса. Через 15 секунд — дашборд: 12 классов с нарушениями, конкретные проблемы, ссылки на пункты нормативов. Скачивает PDF, показывает директору.
**Фаза 2. «Оживление» расписания (Trial, день 1)**
* Регистрируется на 14-дневный trial. Проходит по 12 нарушениям: система предлагает исправления, Елена принимает или корректирует. За 30 минут — все красные маркеры стали зелёными. «Раньше на это уходил целый день.»
**Фаза 3. Экстренная замена (Trial, неделя 2)**
* Вторник, 7:20. Учитель физики на больничном, 4 урока. Елена отмечает отсутствие → за 5 секунд план замен с объяснениями → подтверждение в один клик → карточка замен в чат Сферума и на стенд. 3 минуты вместо 45.
**Фаза 4. ИУП для 10-х классов (Октябрь)**
* Директор запускает профильное обучение: 4 класса, 112 учеников. Елена создаёт форму выбора, ученики заполняют за неделю. Нажимает «Сформировать группы» — за 40 секунд из 30+ уникальных наборов формируются 9 групп. 104 ученика без конфликтов, для 4-х — варианты решения. Запускает генерацию расписания: 3 минуты. Раньше: 2–3 недели.
**Фаза 5. Конверсия**
* На 12-й день trial’а Елена получает письмо с персональной статистикой: 12 исправленных нарушений, 7 замен за среднее время 4 минуты, ИУП для 112 учеников. Подписка — 14 400 ₽/год. Директор сравнивает с лицензией Ректора (17 000 ₽/год) без замен, без ИУП, без веба. Решение очевидно.
